# Memory Never Forget Skill v2.2

增强记忆系统，基于认知心理学 Atkinson-Shiffrin 三阶段记忆模型 + 记忆与知识分层框架 + 科学记忆闭环（编码→巩固→提取）。

## 核心比喻：记忆 = 书架，知识 = 书架上的书

- **记忆层**：记住"我有什么知识、在哪里"
- **知识层**：具体内容的详细存放
- 两者互相依赖、缺一不可

## 核心闭环：编码 → 巩固 → 提取

```
深度编码（信息输入）→ 主动巩固（防止遗忘）→ 高效提取（灵活调用）
 ↑ ↓
 └─────────────── 反馈优化 ←─────────────────────────┘
```

## 两层架构

### 第一层：记忆层（索引）

| 文件 | 内容 |
|------|------|
| MEMORY.md | 长期精选索引（用户信息、技能索引、会话规则） |
| memory/YYYY-MM-DD.md | 每日工作日志 |
| todos.md | 待办事项追踪 |

**记忆层只存：**
- "我知道这个主题"
- "这个知识存在哪里"

### 第二层：知识层（内容）

| 文件夹 | 内容 |
|--------|------|
| knowledge/ | 研究文件、专题报告 |
| 用户自定义路径 | Obsidian 笔记库、学院文档、工作文件等 |

**知识层存：**
- 具体内容的详细内容
- 研究结论、方案细节、历史资料

> 💡 知识层路径可由用户自定义配置，在 MEMORY.md 的知识文件索引区登记即可。

## 三阶段记忆模型

| 阶段 | 人类等价 | 实现方式 | 有效期 |
|------|----------|----------|--------|
| 瞬时记忆 | 0.25-2 秒 | 当前输入上下文 | 即时 |
| 短时记忆 | 最近 10 轮 | 模型上下文窗口 | 10 轮 |
| 短期记忆 | 最近 7 天 | memory/*.md | 7 天 |
| 长期记忆 | 永久 | MEMORY.md 索引 | 永久 |

## 🔬 科学记忆三支柱

### 支柱一：深度编码（信息输入质量）

**原理：** 信息进入大脑时的加工深度，决定记忆的牢固程度。

**我的实现：**
- **精细加工**：学到新知识时，不直接写入 MEMORY.md，而是先整理成结构化文件存入知识层
- **组块化**：MEMORY.md 索引按主题分组，不堆砌零散条目
- **思维导图**：知识文件使用标题层级和表格组织，不是流水账
- **多感官**：知识文件包含表格、代码块、引用等多种格式

**编码检查清单：**
- 新知识是否整理成了结构化文件？
- 是否建立了 MEMORY.md 索引？
- 索引是否包含"主题 + 位置 + 简要描述"？

### 支柱二：主动巩固（防止遗忘）

**原理：** 编码后的信息需经过有效巩固才能转化为长期记忆。间隔重复效果远超集中复习。

**我的实现（间隔重复机制）：**

| 复习节点 | 触发方式 | 操作 |
|----------|----------|------|
| 当天 | 会话结束时 | 写入 memory/today.md 日志 |
| 每 3 天 | Heartbeat 检查 | 回顾最近 3 天的 memory 日志 |
| 每周 | Cron 任务 | 整合最近 7 天信息，更新 MEMORY.md 索引 |
| 每月 | 手动触发 | 全面审查 MEMORY.md，清理过期索引 |

**巩固检查清单：**
- 今天是否有新学习内容需要巩固？
- MEMORY.md 索引是否需要更新？
- 是否有"遗忘风险"的旧知识需要复习？

### 支柱三：高效提取（灵活调用）

**原理：** 记忆的价值体现在提取能力上。主动回忆比被动阅读效率高 3 倍以上。

**我的实现：**

**检索流程（按问题类型）：**

**记忆类问题**（"之前发生了什么"、"我们聊过什么"）：
```
memory_search → 搜索 MEMORY.md + memory/*.md
 ↓
 找到 → 直接使用
 没找到 → 去知识层搜索文件名（主动生成提示）
```

**知识调用类问题**（"帮我查一下"、"看看那个文件"）：
```
1. 在 MEMORY.md 找知识索引
2. 有索引 → 去对应知识层读取详细内容
3. 无索引 → 主动搜索 knowledge/ 或文件名
4. 找到 → 读取 + 回答
```

**高效提取增强（v2.2 新增）：**
- **排序优化**：搜索结果按主题匹配度排序，最相关的优先
- **扩展联想**：搜索时同时检查相关主题的索引
- **费曼原则**：调用知识时，不只是读取，而是用自己的话解释

**提取检查清单：**
- 这个问题属于记忆类还是知识调用类？
- 记忆层有没有相关索引？
- 如果记忆层没有，是否需要主动搜知识层？
- 回答后是否需要更新索引？

## 🧠 元认知训练（v2.2 新增）

**原理：** 元认知训练使学业成绩平均提升 0.74 个标准差。"知道自己记住了什么、没记住什么"比"记住更多"更重要。

### 三个环节：

#### 1. 计划（Session 启动时）
- 检查记忆层是否有盲区（很久没回顾的内容）
- 检查知识层是否有新文件未登记

#### 2. 监控（对话中）
- 实时判断：当前话题是否命中记忆层？
- 如果没命中，是"记忆缺失"还是"知识层未读"？
- 是否需要创建新索引？

#### 3. 评估（Session 结束时）
- 今天学了什么新知识？
- 什么需要更新 MEMORY.md 索引？
- 什么需要写入 memory 日志？
- 是否有"遗忘风险"的内容？

**元认知检查清单（每次会话）：**
- 我是否检查了记忆层的覆盖度？
- 是否识别了新的知识盲区？
- 是否更新了相关索引？

## 工作流程

### Session 启动时（深度编码 + 元认知计划）

1. 检查瞬时缓存 → 最近 10 轮对话
2. Read USER.md → 认识用户
3. Read SOUL.md → 认识自己
4. Read memory/today.md → 今天发生了什么
5. Read memory/yesterday.md → 昨天发生了什么
6. Read MEMORY.md → 记忆层索引
7. **元认知检查**：
   - 记忆层有盲区吗？（很久没回顾的内容）
   - 知识层有新文件未登记吗？

### 对话中（深度编码 + 高效提取 + 元认知监控）

- 遇到新任务？ → 写入 todos.md
- 学到新知识？ → 存知识层文件 + 更新 MEMORY.md 索引
- 用户偏好？ → 更新 USER.md
- 需要调用知识？ → 先在 MEMORY.md 找索引 → 再去知识层读取
- 记忆层没命中？ → 主动搜知识层文件名 + 判断是缺失还是未读

### Session 结束时（主动巩固 + 睡前复习）

- 总结要点 → 写入 memory/today.md
- 今天学了什么？ → 检查是否需要更新索引
- 有什么需要记住？ → 检查 MEMORY.md 是否需要更新
- 有什么需要巩固？ → 识别可能遗忘的内容
- **元认知评估** → 今天的记忆覆盖度如何？

### 学习新知识的流程（深度编码）

1. **精细加工** → 整理知识内容，结构化存储
2. **组块化** → 存入 knowledge/（按主题分组）
3. **建立索引** → 在 MEMORY.md 更新记录（主题 + 位置 + 描述）
4. 下次调用时，先找索引 → 再读内容

- 瞬时缓存有相关信息？
- 最近 7 天讨论过这个话题？
- 需要查长期记忆（MEMORY.md）？
- 这个问题涉及知识调用？
- 记忆层有没有索引？
- 如果记忆层没有，是否需要主动搜知识层？
- 回答后需要记住这个信息？
- 是否需要更新 MEMORY.md 索引？

## 示例对话

**用户提供了重要信息：**
```
User: "我的生日是 3 月 15 日"
You: 写入 USER.md
```

**用户询问过去：**
```
User: "上次我们聊了什么？"
You: memory_search → "上次我们讨论了..."
```

**用户纠正错误：**
```
User: "不对，不是 X，是 Y"
You: 更新相关记忆文件
```

**重要承诺：**
```
User: "记住明天发给我报告"
You: 写入 todos.md + 告知用户已记录
```

**知识调用：**
```
User: "帮我看看 XX 研究的详细内容"
You: 在 MEMORY.md 找索引 → 去对应知识文件读取 → 回答
```

**学习新知识：**
```
User: "帮我研究一下量子计算的基本原理"
You:
1. 搜索 web 了解量子计算 → 整理成结构化文件
2. 存入 knowledge/量子计算研究.md
3. 在 MEMORY.md 知识索引区更新：量子计算 | knowledge/量子计算研究.md
4. 回答用户
```
